# LoRa Car Alarm

Двухблочная система охраны автомобиля на базе LilyGO T3S3 (ESP32-S3) + LR1121.

Блок в машине следит за салоном через радар LD2410B и отправляет зашифрованную тревогу по LoRa. Ручной блок получает пакет и выводит на экран тип угрозы, расстояние до цели и качество сигнала.

---

## Схема системы

```
┌─────────────────────────────┐          LoRa 868 MHz          ┌────────────────────────┐
│        БЛОК В МАШИНЕ        │  ──────── AES-128 ──────────►  │    РУЧНОЙ БЛОК (RX)    │
│                             │                                 │                        │
│  LD2410B ──► ESP32-S3       │                                 │  ESP32-S3              │
│  (радар)     (LR1121 TX)    │                                 │  (LR1121 RX)  OLED     │
│              OLED  Battery  │                                 │               Battery  │
└─────────────────────────────┘                                 └────────────────────────┘
```

## Железо

| Компонент | Назначение |
|-----------|------------|
| LilyGO T3S3 v1.1 | основная плата (ESP32-S3 + LR1121 + OLED + аккумулятор) |
| HLK-LD2410B | 24 ГГц FMCW-радар, детекция человека в салоне |
| ATGM336H | GPS-модуль (зарезервирован, не подключён) |

### Распиновка (T3S3 v1.1)

| Сигнал | GPIO |
|--------|------|
| SPI SCK / MISO / MOSI / NSS | 5 / 3 / 6 / 7 |
| LR1121 BUSY / RST / DIO9 | 34 / 8 / 36 |
| I2C SDA / SCL (OLED) | 18 / 17 |
| Кнопка / OTA | 0 |
| Светодиод | 37 |
| Battery ADC | 1 |
| **LD2410B TX / RX** | **9 / 10 (UART1)** |
| ATGM336H TX / RX (резерв) | 16 / 15 (UART2) |

### Подключение LD2410B

```
LD2410B  →  T3S3
───────────────────
TX       →  GPIO 9
RX       →  GPIO 10
GND      →  GND
VCC      →  3.3V
```

## Работа с устройством через Claude Code

Проект включает кастомный скилл `/micropython` для автоматизации всех операций с устройством.

### Быстрый старт

```
/micropython              — прошить firmware (выбор .bin файла)
/micropython chip         — информация о чипе и flash-памяти
/micropython version      — версия MicroPython на устройстве
/micropython memory       — свободная RAM и место на диске
/micropython deploy       — залить код на TX или RX блок
/micropython logs         — boot-лог (soft reset + захват 25 сек)
/micropython logs 50      — первые 50 строк boot-лога
/micropython logs stream  — стрим serial в реальном времени
```

Скилл автоматически определяет COM-порт устройства по VID `303A` (Espressif).

### Логи

`/micropython logs` делает soft reset через serial (`Ctrl+C` + `Ctrl+D`) и захватывает весь вывод с самой первой строки загрузки. Не использует `mpremote reset`, чтобы не терять начало лога.

`/micropython logs stream` выдаёт готовую команду для запуска в терминале — стрим блокирующий, поэтому его нужно запускать напрямую (`! python -c "..."`), а останавливать `Ctrl+C`.

## Установка прошивки (первый раз, через USB)

```bash
# 1. Стереть flash
python -m esptool --port COM15 erase_flash

# 2. Залить MicroPython
python -m esptool --port COM15 --baud 460800 write_flash 0 ESP32_GENERIC_S3-FLASH_4M-20241129-v1.24.1.bin

# 3. Скопировать файлы прошивки на блок в машине (TX)
mpremote cp main.py lr1121.py crypto.py oled.py battery.py logging.py ssd1306.py ld2410b.py boot.py webrepl_cfg.py secret.key :

# 4. На ручной блок (RX) — то же самое, но без ld2410b.py не обязательно
mpremote cp main.py lr1121.py crypto.py oled.py battery.py logging.py ssd1306.py boot.py webrepl_cfg.py secret.key :
```

> **Важно:** оба блока должны иметь одинаковый `secret.key`.  
> Сгенерируй ключ на первом блоке, скопируй файл на второй:  
> `mpremote cp :secret.key .` → `mpremote cp secret.key :`

## Обновление прошивки по воздуху (WebREPL OTA)

### Подготовка (один раз)

Установи утилиту для загрузки файлов:

```bash
pip install mpremote
# или скачай webrepl_cli.py с https://github.com/micropython/webrepl
```

Пароль по умолчанию: **`ota12345`** — измени его в файле `webrepl_cfg.py` перед первым деплоем.

### Вход в режим OTA

1. **Держи кнопку BOOT** (физическая кнопка на плате, GPIO 0)
2. **Включи питание** (или нажми RESET), не отпуская кнопку
3. **Удерживай 2 секунды** — светодиод мигнёт **6 раз**, затем загорится **постоянно**
4. Устройство подключилось к Wi-Fi и ждёт подключения

```
Нормальная загрузка:  нажал → сразу отпустил → main.py запускается
OTA режим:            нажал → держишь 2 с → LED мигает → LED горит постоянно
```

> Если Wi-Fi недоступен — светодиод мигнёт 9 раз (SOS) и устройство перезагрузится в нормальном режиме.

### Загрузка файлов

Узнай IP-адрес устройства из лога в серийном порту (`[OTA] WebREPL active at 192.168.X.X:8266`) или из роутера.

**Через браузер** — открой `webrepl.html` из репозитория MicroPython WebREPL:

```
ws://192.168.X.X:8266
пароль: ota12345
```

**Через командную строку** (`webrepl_cli.py`):

```bash
# Загрузить один файл
python webrepl_cli.py -p ota12345 ws://192.168.X.X:8266/ main.py :/main.py

# Загрузить несколько файлов (пример скрипта)
for f in main.py lr1121.py crypto.py ld2410b.py; do
    python webrepl_cli.py -p ota12345 ws://192.168.X.X:8266/ $f :/$f
done
```

**Через mpremote** (если поддерживается WebREPL):

```bash
mpremote connect ws:192.168.X.X,ota12345 cp main.py :
```

### Завершение OTA

После загрузки всех файлов нажми **RESET** — устройство загрузится в нормальном режиме.

## Как работает система

### Блок в машине (TX + радар)

```
Включение
    │
    ├── boot.py: кнопка не зажата → normal boot
    │
    ├── Инициализация OLED, радара LD2410B, Wi-Fi/NTP
    │
    └── Главный цикл (не засыпает):
            │
            ├── read_frame() каждые 20 мс
            │       │
            │       ├── Никого нет → OLED "RADAR WATCH / No presence"
            │       │
            │       └── Человек обнаружен!
            │               │
            │               ├── Кулдаун 10 с не истёк → OLED "COOLDOWN Xс"
            │               │
            │               └── Кулдаун истёк:
            │                       Читаем батарею
            │                       Шифруем payload AES-128 CBC
            │                       Отправляем по LoRa (+22 dBm, SF12, 868 МГц)
            │                       OLED "TX ALARM OK" + мигание LED
            │
            └── Кнопка: переключить TX ↔ RX
```

### Payload пакета

```json
{
  "msg": "Alarm!",
  "t":   1744123456,
  "c":   42,
  "v":   3.87,
  "b":   72,
  "ch":  false,
  "ts":  1,
  "dd":  142
}
```

| Поле | Значение |
|------|----------|
| `msg` | тип сообщения |
| `t` | unix-время отправки |
| `c` | номер пакета (счётчик потерь) |
| `v` | напряжение батареи, В |
| `b` | заряд батареи, % |
| `ch` | заряжается ли сейчас |
| `ts` | состояние радара: 0=никого, 1=движение, 2=стоит, 3=оба |
| `dd` | расстояние до цели, см |

Пакет зашифрован: `[IV 16 байт][AES-CBC шифротекст]`.

### Ручной блок (RX)

```
Включение
    │
    ├── boot.py: кнопка не зажата → normal boot
    │
    ├── Инициализация OLED, Wi-Fi/NTP
    │
    └── Главный цикл:
            │
            ├── Ждём пакет 30 с (LoRa RX)
            │       │
            │       ├── Пакет получен:
            │       │       Расшифровываем AES-128
            │       │       OLED показывает тип угрозы, расстояние, RSSI/SNR, время
            │       │       Мигание LED, пауза 5 с
            │       │
            │       └── Таймаут 30 с:
            │               OLED "DEEP SLEEP"
            │               Засыпаем (ext1 wakeup по DIO9 при получении пакета)
            │
            └── Кнопка: переключить RX ↔ TX
```

### OLED ручного блока при тревоге

```
┌────────────────┐
│ SECURE RX OK   │
│ R:-87 S:9.5    │
├────────────────┤
│ Mov 142cm B:72%│
│ Fr 14.04.2026  │
│ 14:32 RX:5 L:0 │
└────────────────┘
```

## Настройка

Все параметры — в начале `main.py`:

```python
WIFI_SSID        = "Fold5"       # имя сети
WIFI_PASS        = "..."         # пароль
TIMEZONE_OFFSET  = 2             # UTC+2

ALARM_COOLDOWN_S = 10            # пауза между повторными тревогами, с
RADAR_POLL_MS    = 20            # период опроса радара, мс
RADAR_OLED_MS    = 1000          # период обновления экрана в ожидании, мс
```

Пароль WebREPL — в `webrepl_cfg.py`:

```python
PASS = "ota12345"
```

## Радиопараметры

| Параметр | Значение |
|----------|----------|
| Частота | 868 МГц |
| Модуляция | LoRa |
| Spreading Factor | SF12 (максимальная дальность) |
| Bandwidth | 125 кГц |
| Coding Rate | 4/8 |
| Мощность TX | +22 дБм |
| Sync Word | 0x12 (приватная сеть) |

SF12 + BW125 даёт до ~15 км в прямой видимости. Типичное время передачи одного пакета — 2–4 секунды.
